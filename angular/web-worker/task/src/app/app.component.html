Instructions

This mini application is solely for demonstration purposes, an example which demonstrates how web workers can make multiple concurrent tasks run in parallel without bogging down.

In one pane the UI of the application is represented by a grid with circles, which is constatly updated with the circles being drawn and reacts to mouse clicks. The second pane will host a long running process, which normally will block the UI thread and prevent the UI from doing its job.


  • Improve responsiveness by moving heavy calculation logic into a web worker calculation script.

    HINT: The webworker.service is already pre-prepared to execute the above script logic.
